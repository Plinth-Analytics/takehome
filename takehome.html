  • Write your code in R, and use any packages you’d like.

  • The quality of your code is very important, so make it shine!
  • Focus on quality instead of quantity - a report with a moderate amount of intentional, thoughtful content is much better than one with a lot of disconnected content.
  • When you are finished, submit all of your work in a GitHub repository (can be public or private).
  • Share the repository with nathaniel@plinth.co (my github username is pa-nathaniel).
